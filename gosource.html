GoSource Internship Project
Automatic Documentation and Visualisation of
Cloud Basing Deployments
GoSource & Hamlet
GoSource is a major contributor to the open source Hamlet CI/CD framework which is primarily focused on deployments to cloud based providers (AWS, Azure). GoSource uses Hamlet to generate cloud deployment blueprints based on the products' configuration information which then inform cloud based providers as to how to implement specific deployments.
The Goal
The aim of this project is to investigate and develop facilities to automatically generate documentation and visualisation of deployments as a by-product of the deployment process from the same configuration information used to drive the deployment.
The Solution
We approached this problem in 4 steps:

  1. First of all, we did an environment scan for suitable tools. And eventually we narrowed down to a tool called Diagrams (rendered using Graphviz), which lets you draw the cloud system architecture in Python code.
  2. Use the selected tool to build code samples that generate diagrams with enough detailed information.
  3. Analyze code samples and build a data model in JSON format as an intermediate format to decouple blueprints from diagram generation.
  4. Next with what we have, we started building the actual convertor. I’ve been building the part that translates the file that’s in JSON format to Python codes. In the process I used a templating tool called Jinja2. The GoSource dev team has built the other half that decouples information from hamlet blueprints and polished up the final combined convertor.
